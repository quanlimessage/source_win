Ｍ１プログラムの種類
【M1_1】フォルダ：ベーシック認証
【M1_2】フォルダ：セッション型
【M1_3】フォルダ：クッキー型

・セッションなどがある為（ログインデータがうまく保持できない為。）、基本的にM1での
　0/100対応、静的ページは他社サーバーでプログラムページは自社サーバーに分けた場合の
　対応は不可能と考えた方がいい。

・ＳＳＬではログイン時に入力されたＩＤ，パスワードは
　通常ドメイン時とＳＳＬ時では互換性が無い為、どちらかに処理を統一させる必要性がある
　（GETでデータは渡せたとしてもパスワードがあるので論外となる）

　通常ドメイン時とＳＳＬ時の両方を処理させる場合はエンドユーザーに最低２回は入力してもらうことになる。
